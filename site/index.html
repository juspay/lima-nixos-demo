devbox
GitHub Install Lima Start Script
Prebuilt NixOS on Lima

NixOS VM, ready to run.

devbox boots a prebuilt Lima image for local development. Your macOS home directory stays outside the VM.

  • Prebuilt images
  • Includes flakes, direnv, gh, just
  • Mounts only /tmp/lima-devbox

Start in one command

Install Lima first. On macOS, run brew install lima. 

curl -fsSL https://juspay.github.io/devbox/start | sh -

Work inside devbox

1Shell

limactl shell --workdir=. devbox

2Clone

mkdir -p ~/code && cd ~/code && git clone ...

3VS Code
  1. Cmd-Shift-P -> Remote-SSH: Settings: set Config File to ~/.lima/devbox/ssh.config.
  2. Cmd-Shift-P -> Remote-SSH: Connect to Host...: pick lima-devbox.
File transfer boundary

Use /tmp/lima-devbox intentionally from either side. Clone repos inside the VM.

Customize CPU, memory, disk, or release tag

By default the starter uses the latest release, auto-sizes CPU, memory, and disk from the host, reuses a running VM, and starts a stopped VM. Use --help for overrides: curl -fsSL https://juspay.github.io/devbox/start | sh -s -- --help.

Why --workdir=.?

devbox does not mount your macOS home directory, so the shell should start in a guest path.
